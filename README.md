# WebRTC KMP — Peer-to-Peer Video Calling

A minimal Kotlin Multiplatform video-calling app that runs the **same Compose UI and call-orchestration logic** on Android and iOS, using WebRTC for media and a tiny Node.js WebSocket server for signaling.

| Layer | Tech |
|-------|------|
| Shared UI | Compose Multiplatform 1.9.3 |
| Shared logic | Kotlin Multiplatform 2.2.21 (`expect`/`actual`) |
| Media | WebRTC — [stream-webrtc-android](https://github.com/GetStream/webrtc-android) on Android, GoogleWebRTC CocoaPod on iOS |
| Signaling transport | Ktor 3 WebSocket client ⇄ Node.js (`ws`) server |
| ICE | Google's public STUN servers (no account required) |

## Project Layout

```
signaling-server/                ~150-line Node.js WebSocket relay
composeApp/
  src/commonMain/
    App.kt                       Router — maps CallState → screen
    CallViewModel.kt             Wires signaling + WebRTC
    SignalingClient.kt           Ktor WebSocket client
    WebRTCManager.kt             expect class (platform contract)
    models/                      CallState, SignalingMessage
    screens/                     LobbyScreen, LoadingScreen, CallScreen,
                                 EndedScreen, FailedScreen
  src/androidMain/               Stream WebRTC + SurfaceViewRenderer
  src/iosMain/                   GoogleWebRTC via cinterop + RTCEAGLVideoView
iosApp/                          Xcode workspace (generated by xcodegen)
```

## Prerequisites

Tools that must be installed on your machine:

| Tool | Why | Install |
|------|-----|---------|
| JDK 17+ | Gradle / Kotlin compiler | `brew install --cask zulu@17`, or use Android Studio's bundled JDK |
| Android Studio | Android SDK + emulators | <https://developer.android.com/studio> |
| Node.js 18+ | Run the signaling server | `brew install node` |
| Xcode 15+ | iOS build | Mac App Store |
| CocoaPods | iOS native deps | `brew install cocoapods` |
| xcodegen | Regenerates `iosApp.xcodeproj` from `project.yml` | `brew install xcodegen` |

Two per-machine files are gitignored and must be created once:

**`local.properties`** at the project root:
```properties
sdk.dir=/Users/<you>/Library/Android/sdk
```

**`gradle.properties`** at the project root:
```properties
android.useAndroidX=true
android.suppressUnsupportedCompileSdk=35
org.gradle.jvmargs=-Xmx4g -XX:+UseG1GC
kotlin.code.style=official
kotlin.apple.xcodeCompatibility.nowarn=true
```

## Run It

### 1. Start the signaling server

```bash
cd signaling-server
npm install
npm start
# → Signaling server running on ws://localhost:3000
```

Leave this running. If port 3000 is already in use: `lsof -ti:3000 | xargs kill -9`.

### 2. Set `JAVA_HOME` to Android Studio's bundled JDK

This avoids the periodic "Java X is incompatible with Kotlin Y" breakage:

```bash
export JAVA_HOME="/Applications/Android Studio.app/Contents/jbr/Contents/Home"
```

### 3a. Run on Android

Boot two emulators (or one emulator + one device on the same Wi-Fi), then:

```bash
./gradlew :composeApp:installDebug
```

Open the app on both, grant camera + microphone, enter the same room ID, tap **Join Room** on each — the call connects automatically.

### 3b. Run on iOS

The Xcode project is generated; regenerate any time `iosApp/project.yml` or the Podfile changes:

```bash
cd iosApp && xcodegen generate              # iosApp.xcodeproj
cd .. && ./gradlew :composeApp:podInstall    # .podspec + pod install
open iosApp/iosApp.xcworkspace               # ALWAYS .xcworkspace, not .xcodeproj
```

Build & run (⌘R). Pick a **(Rosetta)** simulator from the destination dropdown — GoogleWebRTC 1.1.x lacks an arm64-simulator slice, so the simulator build runs the x86_64 binary under Rosetta on Apple Silicon. Real iPhones are native arm64 and Just Work.

If the **(Rosetta)** option isn't visible:
```bash
defaults write com.apple.iphonesimulator "LoadRosettaSimulators" -bool YES
```
…then restart Xcode.

## Signaling Server URL

The default in [`CallViewModel.kt`](composeApp/src/commonMain/kotlin/com/webrtckmp/CallViewModel.kt) is `ws://10.0.2.2:3000`.

| Where | URL |
|-------|-----|
| Android emulator → host machine | `ws://10.0.2.2:3000` *(default)* |
| Real Android device on same Wi-Fi | `ws://<your-LAN-IP>:3000` |
| iOS simulator | `ws://localhost:3000` |

## Architecture

```
                  ┌──────────────────────────────────┐
                  │           commonMain             │
                  │                                  │
                  │  App.kt ─── CallViewModel.kt     │
                  │  (router)         │              │
                  │   screens/   ┌────┴────┐         │
                  │              │         │         │
                  │     SignalingClient  WebRTCManager (expect)
                  │      (Ktor WS)                   │
                  └────────┬──────────────┬──────────┘
                           │              │
              ┌────────────┘              └────────────┐
              ▼                                        ▼
       androidMain                                 iosMain
   WebRTCManager.android.kt                  WebRTCManager.ios.kt
   (org.webrtc.* — Stream fork)              (cocoapods.GoogleWebRTC.*)
```

Each layer has one job:
- Changing the signaling transport → only touch `SignalingClient.kt`
- Changing media behaviour → only touch `WebRTCManager`
- Adding a new screen → one new file in `screens/` plus a branch in `App.kt`

The call lifecycle is modelled as a sealed `CallState`:
```
Idle → Connecting → WaitingForPeer → Negotiating → InCall
                                         ↘
                                          Failed / Ended
```

## Troubleshooting

| Symptom | Cause / Fix |
|---------|------------|
| `Unable to locate a Java Runtime` during Xcode build | Xcode build phases don't inherit your shell's `JAVA_HOME`. The Podfile's `post_install` injects it as a build setting — re-run `pod install` if you changed JDKs |
| `Disconnected from signaling server` on Android | `android:usesCleartextTraffic="true"` missing, or the signaling server isn't running, or `ws://10.0.2.2:3000` is unreachable (check the URL table above) |
| `IllegalStateException: Already initialized` | A `SurfaceViewRenderer.init()` was called twice — `init()` belongs in the `AndroidView` factory, not in `attachLocalRenderer` |
| Simulator marked "build only" in Xcode | GoogleWebRTC's arm64-simulator slice is broken — use a **(Rosetta)** simulator or a real device |
| Android 15 device rejects install / 16KB warning | Make sure `stream-webrtc-android` is ≥ 1.3.x and `jniLibs { useLegacyPackaging = false }` is set |
